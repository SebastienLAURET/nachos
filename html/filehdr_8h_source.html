25 // The following class defines the Nachos "file header" (in UNIX terms,
26 // the "i-node"), describing where on disk to find all of the data in the file.
27 // The file header is organized as a simple table of pointers to
28 // data blocks.
29 //
30 // The file header data structure can be stored in memory or on disk.
31 // When it is on disk, it is stored in a single sector -- this means
32 // that we assume the size of this data structure to be the same
33 // as one disk sector. Without indirect addressing, this
34 // limits the maximum file length to just under 4K bytes.
35 //
36 // There is no constructor; rather the file header can be initialized
37 // by allocating blocks for the file (if it is a new file), or by
38 // reading it from disk.
